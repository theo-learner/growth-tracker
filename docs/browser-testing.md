
## 1. 온보딩 플로우

### 1-A. 정상 온보딩

| 단계 | 행동 | 기대 결과 |
|------|------|-----------|
| Step 1 | 이름 "지호", 만 5세, 남아 선택 | "다음 단계 →" 활성화 |
| Step 1 | "다음 단계 →" 클릭 | Step 2로 전환, 프로그레스 바 33% |
| Step 2 | Q1: "처음엔 좀 살피다가..." 선택 | 카드 primary 강조 |
| Step 2 | Q2/Q3 선택 (선택 안 해도 됨) | 다중 선택 가능 |
| Step 2 | "다음 단계 →" 클릭 | Step 3로 전환, 프로그레스 바 67% |
| Step 3 | "검사 없이 시작하기" 클릭 | 메인 앱 진입 |
| 결과 | 홈 탭 표시 | 샘플 데이터 자동 로드됨 |

**검증 포인트**
- [ ] 홈 탭에 "지호" 이름 표시
- [ ] D+1 뱃지 표시
- [ ] "이번 주 발달 상태" 5개 영역 표시
- [ ] K-DST 체크리스트 표시

## 2. 홈 탭

### 2-A. 이번 주 발달 상태

**위치:** 홈 탭 → AI Daily Note 아래

| 테스트 | 행동 | 기대 결과 |
|--------|------|-----------|
| 종합 배너 | 화면 진입 | 😊/💪/🤗 이모지 + 한줄 메시지 표시 |
| 영역별 카드 | 5개 영역 확인 | 언어이해/시공간/유동추론/작업기억/처리속도 |
| 백분위 표시 | 각 카드 우측 | "상위 N%" 형식 (40~97% 범위) |
| 해석 메시지 | 각 카드 확인 | ✅또래 상위 N% / ⚡또래 중상위권 / ⚠️또래 하위권 |
| 트렌드 | 각 카드 확인 | trending_up(파랑) / flat(회색) / down(빨강) + 증감 점수 |
| 백분위 검증 | 샘플 데이터(72~82점) | "상위 5~25%" 범위 내 표시 확인 |

### 2-B. K-DST 발달 체크리스트

**위치:** 홈 탭 → 이번 주 발달 상태 아래

| 테스트 | 행동 | 기대 결과 |
|--------|------|-----------|
| 초기 상태 | 화면 진입 | "0/N 달성" + 빈 프로그레스 바 |
| 항목 클릭 | 체크박스 탭 | 체크 표시 + 글자 취소선 + primary 배경 |
| 재클릭 | 체크된 항목 탭 | 체크 해제, 원래 상태 복원 |
| 진행도 | 3개 중 2개 체크 | 프로그레스 바 67%, "2/3 달성" |
| 전체 완료 | 모든 항목 체크 | 🎉 "이 시기 발달 과업을 모두 달성했어요!" |
| 새로고침 | 페이지 리로드 | 체크 상태 유지 (localStorage) |
| 도메인 배지 | 각 항목 확인 | 대근육(파랑)/소근육(보라)/인지(청록)/언어(스카이)/사회성(주황)/자조(분홍) |

### 2-C. 기록 추가 - RecordSheet

#### 활동 기록

| 테스트 | 행동 | 기대 결과 |
|--------|------|-----------|
| 바텀시트 열기 | "활동" 버튼 클릭 | RecordSheet 위에서 슬라이드 |
| 카테고리 선택 | "퍼즐" 클릭 | 파란 배경으로 강조 |
| 시간 입력 | "30" 입력 | 숫자 입력 |
| 기록 완료 | "기록 완료" 클릭 | CompletionScreen 표시 |
| 완료 화면 | 확인 | ✅ 아이콘 + "기록 완료!" + 팁 카드 + 다음 활동 |
| 자동 닫힘 | 3초 대기 | 바텀시트 닫힘, 타임라인 업데이트 |
| 수동 닫힘 | "확인" 버튼 클릭 | 즉시 닫힘 |

**난이도 입력 (활동별 정량 추적)**
| 테스트 | 행동 | 기대 결과 |
|--------|------|-----------|
| 퍼즐 난이도 프리셋 | 퍼즐 선택 → 난이도 영역 표시 | [36조각] [72조각] [108조각] [180조각] 버튼 |
| 프리셋 선택 | "108조각" 클릭 | primary 배경으로 강조 |
| 직접 입력 | "직접입력" 클릭 → 숫자 입력 | 입력 필드 노출, 임의 숫자 저장 |
| 블록 난이도 | 블록 선택 | [5층] [10층] [20층] 버튼 표시 |
| 카테고리 변경 | 퍼즐→블록 전환 | 이전 난이도 초기화 |
| 타임라인 확인 | 저장 후 | "퍼즐 · 108조각 — 30분" 형식 표시 |
| 두 번째 기록 후 | 동일 카테고리 2회 이상 저장 | 타임라인에 "지난번 대비 +36조각" 배지 |

**입력 검증**
| 테스트 | 행동 | 기대 결과 |
|--------|------|-----------|
| 카테고리 없이 저장 | 카테고리 미선택 → 기록 완료 | 저장 안 됨 (return) |
| 시간 음수 | -1 입력 | alert "0~600분 사이로 입력해주세요" |
| 시간 초과 | 601 입력 | alert "0~600분 사이로 입력해주세요" |

| 테스트 | 행동 | 기대 결과 |
|--------|------|-----------|
| 이미지 선택 | 파일 선택 → 미리보기 표시 | 파일명 + 이미지 프리뷰 |

| 분석 유지 | 새로고침 후 | aiAnalysis 결과 localStorage에 저장됨, 재표시 |
| API 없을 때 | API 키 미설정 환경 | 프리셋 긍정 분석 텍스트 표시 (fallback) |

### 2-E. 자연어 기록 (QuickInput)

| 테스트 | 입력 | 기대 결과 |
|--------|------|-----------|
| 단순 활동 | "퍼즐 30분 했어" | activity: 퍼즐, 30분 (신뢰도 ≥0.9) |
| 독서 | "구름빵 읽었어" | reading: 구름빵 (신뢰도 ≥0.9) |
| 질문 | "'왜 달이 있어?' 물어봤어" | question: "왜 달이 있어?" (신뢰도 0.9) |
| 복합 | "퍼즐 30분 하고 구름빵 읽었어" | activity + reading 2건 파싱 |
| 감정 | "오늘 엄청 행복해했어" | emotion: 😊 행복 |
| 애매한 입력 | "놀았어" | activity: 기타 (신뢰도 0.3, 🔴 배지) |
| 빈 입력 | 공백만 | 기록 버튼 비활성 |

**ParseConfirmSheet 검증**
- [ ] 신뢰도 < 0.5: 빨간 테두리 + "수정 필요" 배지
- [ ] 0.5~0.7: 주황 테두리 + "확인 필요" 배지
- [ ] ≥0.7: 일반 테두리
- [ ] 각 항목 인라인 편집 가능
- [ ] "모두 기록하기 (N건)" 버튼으로 일괄 저장

### 9-A. 디자인 시스템 일관성

**확인 방법:** DevTools → Elements 패널에서 계산된 CSS 값 직접 확인

#### 컬러 토큰

| 항목 | 위치 | 기대값 |
|------|------|--------|
| Primary 버튼 배경 | `btn-primary` 클래스 | `#2bbdee` (Sky Blue) |
| 앱 배경 | `body`, `.bg-surface` | `#f6f7f8` |
| 카드 배경 | 흰 카드 영역 | `#ffffff` |
| 텍스트 주요 | 제목, 본문 | `#1a1a2e` (dark-gray) |
| 텍스트 보조 | 설명, 레이블 | `#6b7280` (mid-gray) |
| 해석 메시지 | "잘 하고 있어요" | `text-primary-700` ≥ `#0a78a3` |

- [ ] 탭 아이콘 활성 상태: Sky Blue (#2bbdee)
- [ ] 탭 아이콘 비활성 상태: mid-gray (#6b7280)
- [ ] 프로그레스 바 채워진 색: primary
- [ ] 쿠팡 버튼: 노란 계열 그라디언트 (파트너스 브랜딩 유지)

#### 타이포그래피

| 요소 | 기대 스타일 |
|------|------------|
| 탭 제목 (예: "지호를 위한 맞춤 추천") | `text-xl font-bold` |
| 섹션 소제목 (예: "이번 주 발달 상태") | `text-base font-bold` |
| 카드 본문 설명 | `text-sm`, `leading-relaxed` |
| 배지·라벨 | `text-[11px] font-bold uppercase tracking-wide` |
| 보조 텍스트 | `text-xs text-mid-gray` |

- [ ] 줄바꿈 없이 한 줄에 들어가야 할 텍스트가 잘리지 않음 (line-clamp 동작)
- [ ] 모든 폰트가 한글 표시 (Pretendard 적용 확인)
- [ ] 이모지 크기가 주변 텍스트와 시각적으로 조화로움

#### 간격·레이아웃

- [ ] 카드 내부 패딩: `p-4` (16px) 또는 `p-3` (12px) 일관 적용
- [ ] 섹션 간 간격: `space-y-8` (32px) 유지
- [ ] 좌우 여백: `px-4` (16px) 일관 적용
- [ ] 카드 모서리: `rounded-xl` (12px) 일관 적용
- [ ] 상단 sticky 헤더와 콘텐츠 겹침 없음

### 9-C. 인터랙션 & 애니메이션

#### 바텀시트 (RecordSheet)

- [ ] 열릴 때: 위에서 슬라이드 인 (부드러운 전환)
- [ ] 백드롭: `backdrop-blur-md` 배경 블러 + 반투명 처리
- [ ] 핸들 바: 상단 중앙에 작은 회색 막대 표시
- [ ] 백드롭 탭 시: 바텀시트 닫힘 (onClose 호출)
- [ ] `max-h-[70vh] overflow-y-auto`: 콘텐츠 길어도 스크롤 가능
- [ ] 완료 화면 전환이 자연스럽게 (phase 전환 시 끊김 없음)

#### 버튼 인터랙션

| 버튼 종류 | 기대 인터랙션 |
|-----------|--------------|
| `btn-primary` | `active:scale-[0.97]` + 그림자 축소 |
| 탭 전환 버튼 | 즉각 탭 전환, 0.15s 내 |
| 새로고침 버튼 (추천) | `animate-spin` 스피너 + `disabled:opacity-50` |
| 카테고리 선택 버튼 | primary 배경으로 즉시 전환 |
| K-DST 체크 항목 | primary 배경 + 취소선 + 부드러운 전환 |

#### 프로그레스 바 (K-DST 체크리스트)

- [ ] 체크 추가 시: 바 너비 애니메이션으로 증가 (`transition-all`)
- [ ] 완료(100%) 시: 🎉 메시지 등장
- [ ] 반올림 없이 정확한 % 반영 (예: 2/3 → 66.7%)

#### 상태 전환

- [ ] 로딩 상태: 스피너 or `animate-pulse` skeleton이 자연스럽게 표시
- [ ] 빈 상태 → 데이터 로드: 레이아웃 jump 없이 전환
- [ ] CompletionScreen 3초 카운트다운 후 자동 닫힘 시 화면 튀지 않음

### 9-D. 접근성 (Accessibility)

**확인 방법:** DevTools → Lighthouse → Accessibility 탭, 또는 axe DevTools 확장

#### 색상 대비 (WCAG AA 기준: 최소 4.5:1)

| 텍스트 | 배경 | 판정 |
|--------|------|------|
| `#1a1a2e` (dark-gray) on `#ffffff` | 흰 카드 | ✅ 매우 높음 |
| `#0a78a3` (primary-700) on `#e0f5fc` (primary-50) | 배지·배너 | ✅ ≥4.5:1 확인 필요 |
| `#6b7280` (mid-gray) on `#ffffff` | 보조 텍스트 | ⚠️ ≥4.5:1 확인 필요 |
| 흰 텍스트 on `#2bbdee` (primary) | 버튼 | ✅ 일반적으로 충분 |

- [ ] Lighthouse Accessibility 점수 ≥ 90
- [ ] `mid-gray` 텍스트가 작은 글씨 (12px 이하)에서 대비 부족하지 않음

#### 키보드 접근성

| 시나리오 | 행동 | 기대 결과 |
|----------|------|-----------|
| 탭 탐색 | Tab 키로 탐색 | 포커스 링 표시, 탭 순서 논리적 |
| 바텀시트 닫기 | Esc 키 | 닫힘 (현재 미구현 시 기록) |
| 버튼 활성화 | Enter / Space | 클릭 동작 실행 |
| 바텀시트 열기 | 기록 버튼 포커스 → Enter | RecordSheet 열림 |

#### ARIA & 시맨틱

- [ ] 바텀시트: `role="dialog" aria-modal="true" aria-labelledby="record-sheet-title"` 적용 확인
- [ ] 백드롭: `aria-hidden="true"` 적용 확인
- [ ] 이미지: `alt` 텍스트 의미 있게 작성 (상품명, 활동명)
- [ ] 아이콘만 있는 버튼 (새로고침 🔄): `aria-label="추천 새로고침"` 확인
- [ ] 닫기 버튼: `aria-label="닫기"` 확인
- [ ] 진행도 표시: `aria-label` 또는 시각 외 텍스트로 N/M 달성 전달

| 체크리스트 항목 | `py-3` → ≥44px |
| "쿠팡에서 보기" | `px-3 py-1` → ⚠️ 작을 수 있음, 실측 확인 |
| 설정 ⚙️ 아이콘 | `w-8 h-8` = 32px → ⚠️ 터치 패딩 확인 |

- [ ] 작은 버튼들이 실수로 눌리지 않도록 충분한 간격 확보
- [ ] 스크롤 중 의도치 않은 버튼 클릭 없음
- [ ] 두 손가락 줌인 비활성 (viewport `user-scalable=no` or max-scale=1 확인)

#### 스크롤 동작

- [ ] 바텀시트 내부 스크롤 vs 화면 스크롤 구분 명확
- [ ] 홈 탭: 세로 스크롤 부드럽게 (`-webkit-overflow-scrolling: touch`)
- [ ] 가로 스크롤 영역 (Quick Logs, 영역 필터 칩): 가로 스와이프 자연스럽게 동작
- [ ] iOS Safari: 하단 세이프 에어리어 침범 없음 (탭 바 위치)

### 9-F. 빈 상태·로딩 상태 시각 품질

#### 빈 상태 (Empty States)

| 위치 | 확인 항목 |
|------|----------|
| 홈 타임라인 빈 상태 | 아이콘 + 텍스트 중앙 배치, 배경과 자연스럽게 어울림 |

| K-DST 미선택 | 체크 0/N 상태 → 프로그레스 바 0% 가시적 |

- [ ] 빈 상태 카드 높이가 너무 짧지 않음 (`p-8` 적용 확인)
- [ ] 빈 상태 텍스트가 사용자를 다음 행동으로 안내함

#### 로딩 상태

| 상황 | 기대 표현 |
|------|----------|

| 추천 새로고침 | 🔄 스피너 `animate-spin` |
| 차트 lazy load | 탭 전환 시 짧은 로딩 → 차트 등장 |

- [ ] 로딩 중 버튼 비활성 (`disabled:opacity-50`) + 재클릭 방지
- [ ] 로딩 완료 후 레이아웃 shift 없음 (CLS 최소화)
- [ ] 로딩 상태가 2초 이상 지속될 때 사용자가 인지 가능

### 9-I. 해석 색상 UX 일관성

앱 전체에서 해석 3단계 색상이 동일하게 사용되는지 확인:

| 수준 | 트리거 조건 | 색상 코드 | 사용 위치 |
|------|-------------|-----------|----------|
| good (✅ 또래 상위 N%) | 백분위 ≥ 70 | `bg-primary-50 text-primary-700` | 홈 영역 카드, 추천 탭 |
| normal (⚡ 또래 중상위권) | 백분위 40~69 | `bg-amber-50 text-amber-700` | 동일 |
| attention (⚠️ 또래 하위권) | 백분위 < 40 | `bg-rose-50 text-rose-600` | 동일 |

- [ ] 홈 탭 영역별 카드와 추천 탭 집중 영역 카드의 색상 체계 일치
- [ ] 같은 백분위에 다른 색이 나오지 않음 (≥70 → primary, 40-69 → amber, <40 → rose)
- [ ] 해석 이모지(✅ ⚡ ⚠️)와 색상이 함께 표시되어 색맹 사용자도 인지 가능
